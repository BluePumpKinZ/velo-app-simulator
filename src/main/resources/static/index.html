Simulator Velo

The simulator can run 2 types of simulations: file and random. In a file simulation, the events are read from a CSV file. For each event, this file contains a line with the event data and a delay. The delay (in millis) determines how long to wait after the previous event. The simulator reads this file and executes the necessary actions (API calls or putting messages on the queue).

A random simulation is intended for e.g. performance tests. These are not read from a file but randomly generated according to default settings read from a configuration file. are read
