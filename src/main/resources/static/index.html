Simulator Velo

De simulator kan 2 types van simulaties draaien: file en random. In een file simulatie USID9 worden de events (zie bovenstaande tabel) uitgelezen uit een CSV bestand. Hierin staat per event een lijn met de event data en een delay. De delay (in millis) bepaalt hoe lang er na het vorige event moet gewacht worden. USID10 De simulator leest deze file en voert de nodige acties uit (API calls of messages op de queue zetten).

Een random simulatie is bedoeld voor bv. performance testen. USID12 Deze worden niet uit een file gelezen maar random gegenereerd volgens default instellingen die uit een configuratiebestand worden gelezen
